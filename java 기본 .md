🚦 디버깅할 때 코드 실행 순서(= 실제 JVM 실행 순서)
🔥 핵심 규칙 딱 3개

main 메서드부터 시작한다

위에서 아래로 한 줄씩 실행한다

메서드를 만나면 → 그 메서드 안으로 들어가서 실행한다

이걸 “스택(Stack) 프레임 구조”로 진행해.

```📌 예시 코드로 흐름을 눈으로 따라가보자
public class Main {
    public static void main(String[] args) {
        int a = 10;              // (1)
        int b = add(a, 5);       // (2) 메서드 호출 → add로 점프
        System.out.println(b);   // (5)
    }

    public static int add(int x, int y) {
        int result = x + y;      // (3)
        return result;           // (4) 메인으로 돌아감
    }
}
```
🧭 디버깅 실제 흐름
1) (main 시작) JVM이 main()을 호출

➡️ 스택에 Main 프레임이 딱 생김

2) int a = 10 실행

디버깅 커서(노란 화살표)가 여기 있으면
👉 이 줄이 실행됨

a 변수(10)가 스택에 저장됨

3) int b = add(a, 5); 도착

여기서 중요한 포인트:

메서드를 만나면 → JVM은 그 메서드로 이동한다(점프!)

디버깅에서는 보통:

Step Into(F5) → add() 안으로 들어감

Step Over(F6) → add() 실행만 하고 결과만 받음

4) add() 안으로 들어감

add 메서드 프레임이 스택에 새로 생김

main()
  └ add()

5) add 내부 실행
int result = x + y;   // (3)
return result;         // (4)


여기까지 실행되면 add()는 끝남 → 스택에서 제거됨
그리고 결과값(result)이 main으로 다시 전달됨

6) main으로 돌아와서 실행 재개

add가 끝나면 다시 main의 다음 줄로 이동함:

System.out.println(b);

7) 프로그램 종료

main도 끝나면 → 스택 프레임이 비게 되고 프로그램 종료

✔️ 요약 흐름표
실행순서	동작
1	JVM → main() 호출
2	main 안에서 한 줄씩 실행
3	메서드 호출 만나면 → 그 메서드로 이동
4	메서드 끝나면 → 원래 위치로 복귀
5	main 끝나면 프로그램 종료
🧠 디버깅 시 “이해해야 할 핵심 4개”
✔ 1. 메서드는 “들어갔다 → 나왔다”로 보면 된다

→ 진짜 함수 안으로 들어갔다가, return 때 다시 돌아옴

✔ 2. 각 메서드는 “스택에 따로 공간”을 만든다

메서드가 호출될 때마다
새로운 스택 프레임이 생김

메서드 끝나면 그 프레임 사라짐

✔ 3. 참조변수는 “주소”를 들고 이동한다

디버깅 중에 객체를 보면 → 주소값이 유지되는 거임

✔ 4. 디버깅은 “한 줄씩 읽기”가 아니라

👉 “스택이 어떻게 열리고 닫히는지”를 보는 것이다.

🌟 완전 쉬운 비유

메서드 호출은 “문을 열고 방에 들어갔다가, 일이 끝나면 다시 나오는 것”

main 방에서 add 방으로 들어감

할 일 하고 return 하면 → 다시 main 방으로 돌아옴
